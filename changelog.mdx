<Update label="2024-10-15" description="v2.1.0" tags={["feature", "bugfix"]}>
## New Features

- Added support for up to 200 micro-tilts on the Osprey Tilt String (`OTS`), enabling higher resolution monitoring for large-scale diaphragm walls.
- Introduced real-time vibration threshold alerts via the API endpoint `POST /v2/sensors/{sensorId}/alerts`.

## Improvements

- Enhanced data streaming performance by 30% for In-Place Extensometer (`IPX`) readings.
- Updated dashboard visualizations with customizable tilt string profiles.

## Bug Fixes

- Resolved calibration drift issue in `OTS` firmware under high-vibration conditions (railway applications).
- Fixed intermittent connection drops in serial sensor bus for deployments exceeding 100m.

</Update>

<Update label="2024-09-20" description="v2.0.0" tags={["feature", "breaking"]}>
## New Features

- New API endpoint `GET /v2/tilts/{stringId}/profile` for retrieving full tilt string measurement profiles.
- Added webhook support for `OTS` and `IPX` event notifications to `https://your-webhook-url.com/webhook`.

## Breaking Changes

- Firmware version requires `API_KEY` header format update to `Bearer {YOUR_TOKEN}` (previously `Token {YOUR_TOKEN}`).

## Bug Fixes

- Corrected timestamp synchronization errors in multi-sensor deployments.
- Improved trench-embedded sensor resilience to moisture ingress.

</Update>

## Upgrade Process

Follow these steps to apply the latest firmware and software updates to your Osprey Demo sensors.

<Steps>
  <Step title="Check Compatibility" icon="check-circle">
    Verify your device firmware version via the dashboard at `https://dashboard.example.com/devices`.

    <CodeGroup tabs="cURL,JavaScript">
    ````bash
    curl -H "Authorization: Bearer YOUR_TOKEN" \
         https://api.example.com/v2/devices/{deviceId}
    ````

    ````javascript
    const response = await fetch('https://api.example.com/v2/devices/{deviceId}', {
      headers: { 'Authorization': 'Bearer YOUR_TOKEN' }
    });
    const device = await response.json();
    console.log(device.firmwareVersion);
    ````
    </CodeGroup>
  </Step>

  <Step title="Download Update" icon="download">
    Download the latest firmware package from `https://dashboard.example.com/downloads`.
  </Step>

  <Step title="Apply Update" icon="settings">
    Use the serial bus tool to flash the update:

````bash
ots-flash --device /dev/ttyUSB0 --firmware v2.1.0.fw
````
  </Step>
</Steps>
